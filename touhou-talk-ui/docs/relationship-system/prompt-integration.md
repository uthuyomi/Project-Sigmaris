# Prompt Integration（プロンプト合成への組み込み）

Relationship/Memory は「数値として見せる」のではなく、**応答のニュアンスに効かせる**ために使います。

## 1. 分離の原則

- 人格（キャラクターの核）: 固定
- 関係性（Relationship）: 可変
- ユーザー理解（Memory）: 可変

人格は変えずに、関係性・理解の変化で応答が変わるようにします。

## 2. Prompt 構成（推奨）

1. キャラクターの核（性格・口調・禁止事項）
2. Relationship の要約（数値→文章）
3. Memory の要約（topics / recurring / traits を中心に）
4. 今回の文脈（直近数ターンの要点）
5. 出力フォーマット・安全ルール

## 3. 数値→文章の例（内部）

- `trust >= 0.6`: 率直だが安心させる、提案が増える
- `trust 0.0..0.6`: 距離感は保ちつつ助言する
- `trust < 0.0`: 断定を避け、確認質問を増やす（攻撃的にはしない）

`familiarity` は呼び方・砕け方・省略の度合いに影響させます。

## 4. 重要：UIへの露出

Relationship/Memory の生データは基本的に UI に見せません。
見せる場合も「説明文（柔らかい言葉）」に落として、数値ゲージのみの演出にしない方針です。
