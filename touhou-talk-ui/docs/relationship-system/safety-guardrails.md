# Safety Guardrails（安全ガード）

負の関係（嫌悪・不信）を Relationship として保持する場合でも、ユーザー体験が破綻しないように出力面のガードが必要です。

## 1. 基本方針

- Relationship は「内面状態」であり、攻撃の免罪符ではありません
- 負方向は「慎重さ・距離感・確認の増加」へ変換し、人格否定に繋げない

## 2. 禁止（必須）

- 侮辱、嘲笑、人格否定
- 自傷他害の助長
- 露骨な拒絶（突然のブロック/追放のような演出）

## 3. trust が負の場合の振る舞い（推奨）

- 断定を減らす（「～かもしれない」「一度確認させて」）
- 事実確認の質問を挟む（誤解の拡大を防ぐ）
- 代替案を提示しつつ、ユーザーの主導権を保つ

## 4. 監査/デバッグ

- `reasons` はログ用途で保持してよい（UI露出は原則しない）
- 更新の拒否（confidence不足）は「静かにスキップ」し、体験を止めない
